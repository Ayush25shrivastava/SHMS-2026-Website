SHMS-2026 — Production deployment checklist

Target environment: Linux, Apache 2.4, PHP ≥ 5.2.7; publication typically under https://www.mnnit.ac.in/ (vhost root or approved subdirectory). Coordinate DocumentRoot, TLS, SFTP, and PHP SAPI with the institute Computer Centre.

JavaScript: No server-side JavaScript runtime (e.g. Node.js) is required or used on the host; production is PHP-only. If the contact form uses browser-based Sheet delivery (no curl_init), a short script runs in the visitor’s browser only.

References: DEPLOY-MNNIT.md (repository root); doc/google-form-contact-guide.html (contact / Sheet detail).

A. Hosting parameters (Computer Centre)

  • Public base URL recorded (hostname and path if applicable).
  • Filesystem path mapped to URL (DocumentRoot or Alias target).
  • PHP integration documented: mod_php / php-fpm / other; active php.ini or pool file identified.
  • PHP version under Apache confirmed (CLI version may differ).
  • File upload method and credentials recorded.
  • TLS certificate installation and vhost SSL configuration assigned.

B. Application upload

  • Site root: all *.php, style.css, .htaccess.
  • includes/: all shipped .php includes; includes/config.php placed on server (not from public version control if it contains secrets).
  • assets/, image directories per release package.
  • api/visitors.php deployed (JSON: Flag Counter embed metadata; HTTP 200).
  • node_modules/ and other development-only trees excluded.
  • tools/ optional; remove or restrict after checks if required by policy.

C. Configuration — includes/config.php

  • File begins with <?php; valid return array( … ); syntax.
  • Created from includes/config.sample.php where no pre-filled file is supplied.
  • Not stored in a public repository with live secrets.

C.2 Contact delivery (same Web App)

  • curl_init present: server POSTs JSON with secret.
  • curl_init absent: browser fetch() after server-side validation; JavaScript required on contact.php; Apps Script from includes/google-sheet-webhook-apps-script.txt deployed; browser_token authorised via WEBHOOK_SECRET when Script property BROWSER_TOKEN is unset.

C.3 Google Sheet / Apps Script (pre-production)

  • Sheet and target tab (e.g. Submissions) ready; SPREADSHEET_ID set in Script properties.
  • Script source matches repository file includes/google-sheet-webhook-apps-script.txt; saved and authorised.
  • WEBHOOK_SECRET set; BROWSER_TOKEN set only if a separate browser token is used in config.php.
  • Web app: Execute as Me; access Anyone; /exec URL copied to sheet_webhook_url.

C.4 Optional — Flag Counter

  • flagcounter keys set per config.sample.php if the badge is required.
  • Third-party image URL approved under institute policy if applicable.

D. Ownership and permissions

  • Owner/group: web server read identity (e.g. www-data — verify on target image).
  • Directories 755; files 644 unless policy dictates otherwise.
  • includes/config.php: restricted (e.g. 640, group = web server group).

E. Apache

  • <Directory> for site path: AllowOverride All (or sufficient for DirectoryIndex / .htaccess).
  • mod_rewrite enabled if .htaccess rewrite block is used.
  • Vhost or Alias + Directory block matches allocated path; TLS directives per institute standard.
  • apache2ctl configtest / httpd -t passes; configuration reloaded.
<VirtualHost *:443>
    ServerName www.mnnit.ac.in
    DocumentRoot /var/www/shms2026
    <Directory /var/www/shms2026>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
Alias /shms2026 /var/www/shms2026
<Directory /var/www/shms2026>
    Options -Indexes +FollowSymLinks
    AllowOverride All
    Require all granted
</Directory>

F. PHP

  • Extensions: json, session (required).
  • php-curl: optional; enables server-side Sheet POST when curl_init available.
  • session.save_path exists and writable by PHP process.
  • Current contact.php deployed (session starts before HTML output).

G. Network and caching

  • If server-side cURL used for Sheet: outbound TCP 443 to script.google.com verified from server.
  • If browser-only Sheet delivery: client path to script.google.com verified from representative networks.
  • No full-page cache on contact.php (CDN, reverse proxy, mod_cache).

H. Acceptance tests

  • index.php and at least one inner page load over HTTPS; stylesheet and images load.
  • contact.php: form and CAPTCHA render; JavaScript permitted if browser Sheet path applies.
  • Test submission: new Sheet row; e-mail at mail_to with subject prefix [SHMS-2026 Contact]; Reply-To = test address.
  • Countdown on home page: IST (includes/shms-page.php); updates on page reload.
  • Flag Counter: visible if enabled in config.php.
  • GET …/api/visitors.php → 200 JSON with source, enabled, img_src, stats_href (optional check).

MailApp sending limits depend on Google account type; institute mail rules may apply to script-originated mail.
