Registration: PHP site → Sheet + Drive (no cURL)

Delegates submit on registration.php (HTML form, no JavaScript). registration-submit.php POSTs JSON to the Web App doPost handler. Outbound HTTPS uses file_get_contents with follow_location = 0; on 301–308 the same JSON is POSTed again to Location (no cURL, no stream_socket_client).

Site configuration

In includes/config.php, under registration:

  • Recommended (institute sites): webapp_url = empty; webhook_url = the /exec URL above. registration.php then shows only the embedded PHP form; the server POSTs JSON to Google (no browser fetch to script.google.com, so HTML sign-in pages in the browser are avoided).
  • Optional: webapp_url = same /exec URL to show “Open registration form” (Google-hosted UI). If that page returns HTML instead of JSON on submit, use the PHP-only setup or add php_form_alongside_webapp = true.
  • Optional submit_secret = same string as CONFIG.SUBMIT_SECRET in Apps Script (if you enable it).
  • use_curl = true to send the webhook POST with PHP cURL first (enable extension=php_curl / php_curl.dll); otherwise the stream wrapper is tried first.
  • Production: insecure_ssl = false; set ca_bundle or contact.sheet_webhook_cainfo or tools/cacert.pem.
  • Local Windows: insecure_ssl = true only for testing.

Apps Script

Paste includes/registration-webapp/Code.gs.txt into a single Code.gs file (V8 runtime). You do not need a separate Registration.html in Google: the form is embedded in that file unless you set CONFIG.LOAD_REGISTRATION_FROM_PROJECT_FILE to true and add an HTML file named exactly Registration. doPost reads postData.contents and JSON.parses it (same for JSON posted as Content-Type: text/plain from the embedded form, to avoid a CORS preflight). Opening the /exec URL in a browser (GET) may still show a setup page until doGet is satisfied; the PHP path does not depend on that for submissions.

PHP requirements

  • allow_url_fopen enabled.
  • HTTPS outbound: either (1) OpenSSL + allow_url_fopen for file_get_contents('https://…'), or (2) PHP cURL (curl_init) if OpenSSL is not loaded — the registration handler tries the stream first, then cURL. Use tools/cacert.pem or registration.ca_bundle / curl.cainfo if TLS verification fails.
  • Outbound HTTPS to script.google.com and script.googleusercontent.com.
“The server did not return JSON” on the Google-hosted form means the POST got HTML (e.g. sign-in) or an error page, not {"ok":…}. Fix: Web app deployment → Execute as: Me, Who has access: Anyone (or anyone with the link who can sign in). Open the form only from the live /exec URL — not the library URL or Run/Preview from the editor. In Apps Script, set CONFIG.WEBAPP_EXEC_URL in Code.gs to the same /exec URL so the form still POSTs correctly if getUrl() is empty.
Contact form email (SMTP / mail()) is unrelated to this path; registration only talks to Apps Script over HTTPS.
